Complete API Guide

Documentação integral de todos os módulos e fluxos do sistema.

PROTEGIDO (SANCTUM)

1. Autenticação & Segurança

  • POST /login - Autentica e retorna o Bearer Token.
  • POST /register - Cria conta (role: user ou admin).
  • POST /logout - Invalida token atual.

1. Usuário envia credenciais.
2. Sistema retorna token.
3. O Token deve ser enviado no Header Authorization: Bearer {token}.

Register Payload: 
{
  "name": "Admin User",
  "email": "admin@example.com",
  "password": "password",
  "password_confirmation": "password",
  "role": "admin" // opcional: 'user' ou 'admin'
}
Login Payload: 
{
  "email": "admin@example.com",
  "password": "password"
}
const login = async (credentials) => {
  const { data } = await api.post('/login', credentials);
  localStorage.setItem('api_token', data.token);
};

Configuração: Vá em Auth -> Bearer Token e cole o valor retornado.

CATÁLOGO

2. Categorias

  • GET /categories - Listagem pública.
  • POST /categories - ADMIN ONLY Criar.

Categorias agrupam produtos. Apenas administradores podem gerenciar as categorias do sistema.

Create/Update Payload: 
{
  "name": "Eletrônicos",
  "description": "Smartphones, Laptops e acessórios"
}
const categories = await api.get('/categories');

POST: Requer token de um usuário com role: 'admin'.

CATÁLOGO

3. Produtos

  • GET /products - Listagem pública.
  • POST /products - ADMIN ONLY Criar.

Produtos são vinculados a uma categoria via category_id. O estoque é reduzido no checkout.

Create/Update Payload: 
{
  "category_id": "uuid-da-categoria",
  "name": "iPhone 15 Pro",
  "description": "128GB, Space Gray",
  "price": 8999.90,
  "stock": 50
}
const products = await api.get('/products');

Use Query Params para filtros: ?category_id=...

USER AREA

4. Carrinhos

  • POST /carts - Recuperar carrinho ativo.
  • POST /cart-items - Adicionar produto.

O carrinho é persistente por usuário logado. Salve o ID retornado pelo /carts.

Add Item Payload: 
{
  "cart_id": "uuid-do-carrinho",
  "product_id": "uuid-do-produto",
  "quantity": 1
}
await api.post('/cart-items', { cart_id, product_id, quantity });

Crie um carrinho vazio primeiro via POST /carts.

FINALIZAÇÃO

5. Checkout & Pedidos

  • POST /checkout - Finalizar carrinho.
  • GET /orders - Meus pedidos.

O Checkout limpa o carrinho e gera um Pedido Pendente.

Checkout Payload: 
{
  "cart_id": "uuid-do-carrinho"
}
const order = await api.post('/checkout', { cart_id });

Copie o ID do pedido retornado para usar no pagamento.

LIQUIDAÇÃO

6. Pagamentos

  • POST /payments - Pagar pedido.

Muda status para paid e encerra o ciclo de compra.

Payment Payload: 
{
  "order_id": "uuid-do-pedido",
  "payment_method": "pix" // pix, credit_card ou boleto
}
await api.post('/payments', { order_id, payment_method });

O sistema simula uma transação imediata.
