E-commerce API Guide

A arquitetura e fluxo técnico da sua aplicação.

SANCTUM

1. Autenticação

Gerenciamento de acesso e segurança.

  • POST /login - Autentica e retorna token.
  • POST /register - Cria nova conta.
  • POST /logout - Invalida o token atual.

1. O usuário faz POST em /login.
2. Recebe um token.
3. Armazena no localStorage ou Cookies.
4. Envia no Header em todas as outras requisições.

const api = axios.create({
  baseURL: '/api/v1',
  headers: { Authorization: `Bearer ${token}` }
});

Na aba Auth, selecione Bearer Token e insira o valor retornado pelo login.

PÚBLICO / PROTEGIDO

2. Catálogo (Produtos & Categorias)

Listagem e detalhes de produtos e categorias.

  • GET /categories - Lista todas as categorias.
  • GET /products - Lista produtos (com filtros).
  • GET /products/{id} - Detalhes do produto.

Os produtos podem ser filtrados por category_id para carregar páginas específicas no frontend.

const products = await api.get('/products');
setItems(products.data.data);

Basta fazer um GET simples. Se as rotas estiverem protegidas, não esqueça do Bearer Token.

ESTADO ATIVO

3. Carrinho & Itens

Gerenciamento temporário de itens antes da compra.

  • POST /carts - Cria ou recupera um carrinho.
  • POST /cart-items - Adiciona item ao carrinho.
  • PUT /cart-items/{id} - Altera quantidade.
  • DELETE /cart-items/{id} - Remove item.

O cart_id é a âncora aqui. No frontend, você deve salvar o cart_id do usuário para adicionar produtos a ele.

await api.post('/cart-items', { 
  cart_id: activeCartId, 
  product_id: p.id, 
  quantity: 1 
});

Body esperado no items: { "cart_id": "...", "product_id": "...", "quantity": 1 }

FINALIZAÇÃO

4. Checkout & Pagamentos

Conversão de carrinho em pedido e liquidação.

  • POST /checkout - Cria o Pedido (Order).
  • POST /payments - Processa o valor.

1. Checkout: Valida estoque e zera o carrinho.
2. Payment: Recebe o order_id e muda o status do pedido para paid.

const order = await api.post('/checkout', { cart_id });
await api.post('/payments', { 
  order_id: order.data.id, 
  payment_method: 'pix' 
});

Sequência: Primeiro Checkout, pegue o id retornado, então mande no Payments.

HISTÓRICO

5. Pedidos (Orders)

Acompanhamento de compras realizadas.

  • GET /orders - Lista todos os pedidos do usuário.
  • GET /orders/{id} - Detalhes completos e itens.

Após o pagamento bem-sucedido, o pedido permanece no sistema com status paid. O usuário pode consultar o histórico a qualquer momento.

const myOrders = await api.get('/orders');
const details = await api.get(`/orders/${orderId}`);

Use o GET simples com o ID do pedido no final da URL.
